# Специфікація витягу інформації (IE Policy)

У рамках Лабораторної роботи №4 для датасету NLI (Natural Language Inference) обрано 3 типи полів.

## 1. Поле: QUANTITY (Кількості)
- **Що вважаємо полем (визначення):** Числові значення (цифрами або словами), які вказують на кількість об'єктів чи людей у сцені.
- **Формат на виході (нормалізація):** Нормалізоване ціле число (Integer) або рядок (для "кілька"/"багато"). Наприклад, "дві" -> `2`.
- **Що НЕ вважаємо полем (негативні приклади):** Займенники або вказівні слова, які збігаються з числами. Наприклад: "Один з чоловіків" (тут "один" — займенник).
- **5 типових edge cases:**
  1. Займенники: "Один спить, інший читає" (не вважати QUANTITY).
  2. Вік: "Хлопчик 5 років" (це характеристика, а не кількість об'єктів).
  3. Відсотки/Дроби: "100% людей" (не ціле число, ігноруємо).
  4. Номери/ID: "Будівля 5" (це номер/назва, а не кількість будівель).
  5. Прийменники міри: "Біля 5 людей" (тут "5" - кількість, але треба обережно з прийменником).

## 2. Поле: LOCATION (Локації)
- **Що вважаємо полем (визначення):** Фізичні місця або об'єкти середовища, де відбувається дія. Визначається за словником `locations_ua.txt` + обов'язковий просторовий прийменник ("в", "у", "на", "біля", "по").
- **Формат на виході (нормалізація):** Знайдене слово в нижньому регістрі (наприклад, "траві").
- **Що НЕ вважаємо полем (негативні приклади):** Власні назви міст, які не описують фізичну сцену ("Він поїхав у Київ"), або об'єкти, з якими взаємодіють як з предметами.
- **5 типових edge cases:**
  1. Об'єкт vs Локація: "Сидить на воді" (локація) vs "П'є воду" (об'єкт).
  2. Власні назви: "Вулиця Хрещатик" (ловимо тільки "вулиця").
  3. Частини слів: слово "кавоварка" не повинно спрацьовувати на корінь "кафе" чи "парк".
  4. Метафори: "В думках" (не фізична локація).
  5. Локація без прийменника: "Кімната була порожня" (наше правило це проігнорує для збільшення Precision).

## 3. Поле: COLOR (Кольори)
- **Що вважаємо полем (визначення):** Ознаки кольору об'єктів чи одягу, що базуються на словнику `colors_ua.txt`.
- **Формат на виході (нормалізація):** Знайдене слово в нижньому регістрі.
- **Що НЕ вважаємо полем (негативні приклади):** Слова, які містять корінь кольору, але є іншими частинами мови або прийменниками ("біля").
- **5 типових edge cases:**
  1. Прийменники: "Біля столу" (корінь "біл-", хибне спрацювання, додано анти-правило).
  2. Прізвища/Власні назви: "Пан Чорний" (не колір).
  3. Предмети: "Білок", "чорнило".
  4. Складні кольори: "чорно-білий" (може спійматись як два окремі кольори).
  5. Метафори: "чорний день", "біла заздрість" (не фізичний колір).
