# Лабораторна робота №4: Rule-based Information Extraction

**Тема:** Побудова системи витягу інформації на основі правил (regex + dictionaries) для датасету UA NLI.

### 1. Які 3 типи полів витягуємо
У рамках роботи було обрано сутності, що найчастіше зустрічаються в описах візуальних сцен:
* **QUANTITY** (Кількості): Числові значення (цифрами або словами), що вказують на кількість об'єктів.
* **LOCATION** (Локації): Місця дії (вулиця, парк, басейн тощо).
* **COLOR** (Кольори): Ознаки кольору об'єктів чи одягу.

### 2. Які правила/словники використали
* **Словники коренів (resources/):** Використано словники коренів для кольорів (`colors_ua.txt`) та локацій (`locations_ua.txt`) для обходу проблем із відмінками в українській мові.
* **Контекстні правила (Regex + Prep):** Для поля `LOCATION` застосовано правило «прийменник + слово», що дозволяє витягувати локацію лише тоді, коли вона вказує на місце дії (напр., "на траві"), а не на об'єкт.
* **Нормалізація числівників:** Створено мапінг текстових числівників ("дві", "п'ять") у цілі числа (`int`), що забезпечує уніфікацію даних.
* **Анти-правила (Exceptions):** Впроваджено набір винятків для усунення хибних спрацювань на омонімах. Це було покрито лише частково, для демонстрації. Можливе подальше доповнення винятків.

### 3. Precision по кожному типу поля
Оцінка проведена на основі вручну розміченого Gold Subset (50 текстів).

### 4. Топ-5 edge cases 
1. **Займенник "один":** "Чоловік... бачить ззаду один" — слово витягується як кількість, хоча є займенником.
2. **Частини слів:** Слово "більярд" містить корінь "біл-", що спричиняє хибне спрацювання правила для кольору.
3. **Прикметникові локації:** "на лісовій дорозі" — система витягує "лісовій" як локацію, хоча головним словом є "дорога".
4. **Машинний переклад:** "на біличах" — артефакт перекладу слова *bleachers* (трибуни), що помилково сприймається як колір.
5. **Міра vs Кількість:** "багато снігує" — слово "багато" тут є прислівником міри дії, а не лічильником об'єктів. + помилка перекладу, скорше за все мало б бути "сніжить"

### 5. Що плануєте покращити в наступних лабах
Для підвищення точності планується впровадження **POS-тегів** (Stanza/Spacy), щоб розрізняти числівники від займенників та прикметники від іменників, що дозволить усунути більшість знайдених False Positives.
